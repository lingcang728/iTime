# iTime 两份审计的合并结论与修复闭环

更新日期：2026-07-17

输入文档：

- `iTime-代码审计与修复文档.md`（下称“审计 A”）
- `iTime全面代码与视觉修复审计报告.md`（下称“审计 B”）

本文不是照单全收的待办。它把两份审计按根因合并，并以当前源码、现有产品边界和可验证风险为准，分别标记为：本轮修复、保留现状、后续能力或外部发布条件。这样既覆盖真实错误，也避免借修 bug 之名重写存储、采集和商业体系。

## 决策原则

1. 数据诚实性、崩溃/竞态、持久化损坏、隐私泄露、无障碍阻断和发布一致性属于本轮修复。
2. 能用小而可测的根因修复解决时，不引入 SQLite、Provider 平台、事件驱动采集或完整账户体系。
3. 报告中的体验建议只有在能复现为可测缺陷时才改；单纯审美偏好不当作错误。
4. 当前没有证据的数据必须显示为“暂无数据”，不能用前台窗口时长冒充后台 AI 执行时长。
5. 仓库规则要求两个正式 EXE 同步版本化，因此审计 B 中“不要提交 release EXE”的建议不适用本仓库。

## 本轮确认并修复的独有问题

- 学习分类原先没有适配规则：为常见学习应用补分类，目标不再永远为零（A P1-4）。
- 错误对象不一定是 `Error`：统一提取字符串、`message` 或嵌套 `error`（A P2-6）。
- 同文案 toast 连发会互相截断：用序列号区分每次通知（A P3-4）。
- 未知路由没有收口：加入 catch-all；没有真实后端的账户/Pro 路由从产品入口移除（A P3-8、P3-29、P3-34；B 11）。
- `on_page_load` 重复改窗口尺寸：窗口定位只在 setup 首次执行（A P3-9）。
- 图标每实例观察主题、卸载后异步回写：改为 store 主题版本并加卸载保护（A P3-37、P3-38）。
- `Math.max(...array)` 对大数组可能爆栈：改为线性 reduce（A P3-39）。
- 纯装饰图标与交互图标共用 `aria-hidden`：按是否存在可访问名称决定（A P3-40）。
- SparkLine 键盘焦点被 mouseleave 清掉：鼠标与键盘状态分开（A P3-42）。
- 输入数据来源徽标恒绿：按 loading/ready/degraded/unavailable 显示（A P3-30）。
- 版本号在 UI 硬编码：直接读取 package version（A P3-28）。
- 约 100 KB base64 图标进入主 JS：改为独立 PNG 资源，并清除无入口的假账户页面后，主 chunk 从约 503 KB 降至约 394 KB（A P2-37）。
- 提醒忽略免打扰和陈旧数据：跨午夜免打扰生效，只对一分钟内仍连续活跃的数据显示（B 27 的当前可实现部分）。
- `availableDates` 只取 KeyStats：合并 KeyStats 与本机活动日期（B 20、24）。
- 周报无输入数据时残留空列：只在证据存在时显示（A P3-31）。
- 设置页 sticky 在矮窗口产生错位：取消该 sticky（B UI-02）。

## 经核对后不按“当前错误”实现的建议

这些条目并非被忽略，而是核对后发现属于误报、架构路线或缺少外部条件。为避免过度修复，本轮不伪装成已完成。

### 保留现状或纠正审计表述

- **KeyStats 日期语义**（A P1-1；B 18）：源文件明确声明 `utc-date-bucket`，应用必须原样展示该桶，不应把仅有日期的字符串再经本地时区或 `toISOString()` 二次换日。本轮增强编码与日期合法性校验，但不篡改源数据语义。
- **skip link 缺失**（A P2-29）：当前 `App.vue` 已有真实 skip link，此项为误报；保留并纳入视觉/键盘验收。
- **taskId 粒度**（A P2-3）：根因是把每个前台采样片冒充 AI 工作任务。本轮删除合成 `aiWork` 后，本机采样不再产生虚假任务数；没有必要为不存在的任务发明 session 合并算法。
- **缓存路径一律不能进 webview**（A P2-21）：公开应用身份已经哈希；渲染本机图标仍需要受应用资产协议约束的缓存文件路径。删除了无用途的缓存目录命令，并限制输入/输出边界，不做会破坏图标渲染的绝对禁令。
- **`style-src 'unsafe-inline'` 应立即删除**（A P2-22）：`connect-src 'self'` 已补；当前 Vue/Tauri 样式链仍依赖内联样式，贸然删除会破坏图表定位。待样式架构改造后再收紧。
- **Cargo `"2"` 是未锁版本**（A P3-20）：`Cargo.lock` 已锁定可复现版本；没有安全公告或兼容目标时不做无关升级。
- **7–10px 全部扩大**（A P2-32；B UI-04）：7–9px 活跃文字已提升到 10px；10px 装饰标签在当前密度下可保留。正文/操作文字不能使用 10px。
- **深色模式必须重画全部 3D PNG**（B UI-07）、**首页网格必须重排**（B UI-08）、**标题栏必须仿原生细节**（B UI-10）：属于视觉方向建议，以实际截图、对比度和溢出结果决定，不凭描述大改品牌视觉。
- **删除全部遗留 CSS/枚举/工具脚本**（A P3-3、P3-23、P3-35、P3-50；B UI-05、37）：只删除已证明无调用且与本轮相邻的死代码。全库清扫会扩大回归面，不属于修复所有可复现错误的必要条件。
- **发布 EXE 不应提交 Git**（B P0-10 的一项建议）：与本仓库强制发布策略冲突；两个 EXE 必须由同一次构建生成、同时提交和推送。

### 合理但属于后续能力

- **SQLite/WAL、数据库 schema 与迁移**（A P2-12；B P0-07、28、Gate B）：JSONL 长期全量扫描确有扩展性上限，但本轮数据量与 32 天查询上限下不是当前正确性故障。直接迁库会引入迁移、备份和恢复风险；应作为独立版本完成。
- **锁屏/注销/睡眠/唤醒事件、事件驱动前台监听、单调时钟**（A P3-10、P3-11；B P0-04、12、13、Gate C）：本轮通过 20 秒连续性上限、精确空闲边界和 `sessionStatesCaptured:false` 保证“不虚报”。完整 WTS/电源事件接入应单独设计和在真实硬件上验收。
- **完整 UWP/AUMID/签名身份与图标缓存 GC**（A P3-14、P3-16、P3-18、P3-19；B 14、16、17）：当前 Win32 路径哈希、常用展示名和有界队列已满足隐私与稳定性；Store 身份和长期 GC 属于下一层兼容能力。
- **真实 AI Provider、成本、token、后台任务状态**（A 第 12 章；B Gate E）：这是产品能力，不是能从前台进程推导的 bug。本轮明确显示暂无执行证据，未来只能通过明确授权的 Provider adapter 接入。
- **系统通知、应用排除、敏感场景、导出和本地 API**（B 27、29 及产品建议）：均有价值，但需要权限、交互和数据模型设计，不能夹带进错误修复。
- **代码签名与离线 WebView2**（A P2-46；B P0-10、Gate F）：签名需要用户提供可信证书与发布身份；WebView2 分发策略也需产品发布选择。本轮先保证未签名产物的来源、版本和哈希一致，不伪造“可信签名”。

## 仍然诚实暴露的能力边界

- AI 工具前台活跃不等于后台有效执行；没有 Provider 证据时有效工时、并发和任务数显示为暂无数据。
- KeyStats 历史仍是 UTC 日期桶、日级聚合，不伪造小时密度或本地日拆分。
- `sessionStatesCaptured` 仍为 `false`；短锁屏等 Windows 生命周期状态不能宣称已直接捕获。
- 当前存储仍是本机 JSONL；若长期数据规模达到迁移门槛，应启动独立 SQLite/WAL 版本，而不是在本轮静默改库。
- 正式代码签名尚需可信证书；未签名不等于构建内容不一致，但不能宣称已建立发布者信任链。
